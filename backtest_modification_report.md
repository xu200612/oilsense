## 修改失败原因记录

### 失败1: 训练参数全覆盖式修改
**症状**: max_depth 大改、early_stopping 开关、RF 替换 XGBoost、Ridge+Scaler 等
**原因**: 认为 14 棵树是问题根源，试图通过改训练参数解决"预测平坦"
**结论**: 平坦是 10 日涨跌幅信噪比低的固有属性，不是参数能解决的

### 失败2: 极端层融合系数
**症状**: 在 base_mid 和 extreme_mid 之间加权融合，2026 方向掉到 38%
**原因**: 稀释了极端层的覆盖强度，正常的预测和极端的信号混在一起两头不讨好
**结论**: 极端层应该硬覆盖（hard-switch），不是软融合

### 失败3: EMA 平滑
**症状**: 方向损失 2%，但长连续值期间曲线仍接近直线
**原因**: 平滑只是把离散点连成线，不改变 underlying 值的分布
**结论**: 60/60 唯一值听起来很好，但本质上是视觉欺骗

### 失败4: 全量训练（没有 TRAIN_CUTOFF）
**症状**: 模型看到 2026 封锁期数据，方向从 70% 掉到 30%
**原因**: 训练集见过暴涨后，在正常期学会"看空"防止过涨
**结论**: TRAIN_CUTOFF 必须保留，这是正确的设计

### 失败5: 分层验证集抽样
**症状**: 验证集包含高波动期 → 模型学到 bearish bias → 方向掉
**原因**: 高波动期（新冠、关税）大部分是下跌，模型学到了"恐慌 = 做空"
**结论**: 时序验证集必须按时间顺序，不能混入未来信息

## 常见修复做法总览

| 尝试 | 结果 |
|------|------|
| max_depth=3→5→7 | 无实质改善 |
| early_stopping 开关 | 关→过拟合，开→14棵树 |
| DART booster | 训练慢 5 倍，无改善 |
| RandomForest mid | 预测连续但方向偏 |
| Ridge+Scaler mid | 方差暴涨，MAE 6x |
| EMA 平滑 | 方向掉 2%，无本质改观 |
| 全量训练 | 方向 70%→30% |
| 分层验证 | 学到 bearish bias |
| 极端层硬覆盖 | 封锁期有效，正常期跳变 |
