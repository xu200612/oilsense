# OilSense 系统改进报告

**基准版本**：OilSense-TA-v1.1（2026年3月提交版）  
**改进时间**：2026年5月  
**团队**：Heimdallrs

---

## 一、执行摘要

本报告记录了在提交版本基础上对 OilSense 系统的全面技术改进。改进覆盖五个层面：**Bug 修复、第二层模型优化、回测展示重构、数据层扩展、新功能接入**。部分改进修正了原版中存在但未被发现的逻辑缺陷，另一部分扩展了系统的分析能力。

---

## 二、原版系统技术基准

### 2.1 模型性能（提交版数据）

| 指标 | 原版数值 |
|------|---------|
| Enhanced 方向准确率 | ~58%（285条样本外测试集） |
| Baseline 方向准确率 | ~57% |
| P10-P90 区间覆盖率 | 76%（低波动期81%，高波动期68%） |
| LLM情感因子提升 | +1%（sentiment_score、sentiment_accel进入Top10） |
| 测试集时间范围 | 2024-07 至 2026-01，约285条 |

### 2.2 原版架构关键设计

- **第一层**：XGBoost分位数回归（P10/P50/P90），Enhanced含44特征，Baseline含19特征
- **第二层**：15个历史极端事件库 + 加权余弦相似度匹配
- **触发机制**：`vol_ratio > 2` / `VIX > 30` / `|hormuz_z| > 2` / `(geo_flag > 0 AND gdelt_ci < -4.0)`
- **回测展示**：原始 `pred_baseline_mid`（蓝线）+ `pred_enhanced_mid`（灰虚线）
- **危机窗口验证**：新冠+价格战、俄乌冲突、特朗普关税（三个均在训练集内）

## 三、Bug 修复

### 3.1 【严重】get_integrated_output() 缺少 hormuz_blocked 触发条件

**原版问题**：`get_integrated_output()` 的 `is_extreme` 判断仅依赖 `hormuz_tanker_zscore`（阈值|z|>2.0）。由于 z-score 基于90日滚动均值计算，持续性封锁导致均值自适应下移，z-score 在封锁发生约2周后即从 -3.5 衰减至 -1.0，跌破触发阈值。

**后果**：2026年3月2日至5月31日封锁期（50天）中，仅10天触发了第二层，其余40天使用 XGBoost 原始输出——而 XGBoost 对 $90+ 的油价只会预测均值回归（看空），与实际大幅上涨完全背离。

**修复方案**：
```python
# 增加 hormuz_blocked 直接标志位触发
hormuz_blocked = _to_float(row.get("hormuz_blocked", 0), 0.0)
is_extreme = (vol_r > 2.0 or vix_val > 30 or hormuz_z > 2.0 or
              hormuz_blocked > 0 or          # ← 新增
              (geo_flag > 0 and gdelt_ci < -4.0))
```

`hormuz_blocked` 是 IMF PortWatch 原始数据中的直接封锁标志（通过量低于均值50%时置1），不受 z-score 自归一化影响。封锁期 50 天全部正确激活第二层。

---

### 3.2 【严重】get_extreme_prediction() 内部触发条件与外层不一致

**原版问题**：`get_integrated_output()` 判断为极端期并调用 `get_extreme_prediction()`，但后者内部有**独立的** `is_extreme` 判断，不包含 `hormuz_blocked`。当外层因 `hormuz_blocked=1` 触发、内部 z-score 已衰减至 < 2.0 时，内部判断返回 `activated=False`，直接把 XGBoost 原始输出当作情景匹配结果返回。

**后果**：外层认为第二层在运行，实际上情景匹配从未执行，返回的是 XGBoost 的均值回归预测。

**修复方案**：同步在 `get_extreme_prediction()` 内部添加 `hormuz_blocked > 0` 触发条件，保持两处逻辑一致。

---

### 3.3 【中等】Quantile Crossing：P50 超出 P10-P90 区间

**原版问题**：三个分位数模型独立训练，各自优化不同损失函数，不存在保证 P10 ≤ P50 ≤ P90 的约束。全量数据中约3.2%（51行）出现 P50 超出区间的情况，图表中绿线（综合输出）跑出绿色阴影区外。

**修复方案**：
```python
# 极端期（情景匹配输出）：三值排序
e_low, e_mid, e_high = sorted([ext["pred_low"], ext["pred_mid"], ext["pred_high"]])

# 正常期：区间端点夹住 mid
e_low  = min(e_low,  b_mid)
e_high = max(e_high, b_mid)
```

---

### 3.4 【中等】_infer_trigger_type() 优先级错误导致关税期误分类

**原版问题**：`_infer_trigger_type()` 函数以"霍尔木兹航运异常 OR 地缘+GDELT"为第一优先级判断地缘政治类型。2025年4月特朗普关税冲击期间（VIX=52，油价单周-16%），由于 hormuz_z 仍有一定读数，被错误分类为"geopolitics"，情景匹配优先找到俄乌战争、霍尔木兹封锁等供应中断事件（历史上这些事件后油价上涨），输出看涨信号，与实际需求崩溃（油价下跌）方向相反。

**修复方案**：重写优先级逻辑：
```python
# 1. 宏观冲击优先（VIX极高+油价大跌）——捕捉关税、衰退、金融危机
if vix > 35 and ret_5d < -0.05:
    return "macro"
# 2. 实际封锁信号（最确定的地缘政治指标）
if hormuz_blocked > 0:
    return "geopolitics"
# 3. 航运显著异常（更保守的阈值 2.5，原版2.0误触发较多）
if hormuz_z > 2.5:
    return "geopolitics"
```

改进前后对比：
- 2025-04 关税期（VIX=52，r5=-16%）：原版→"geopolitics"；改进后→"macro" ✓
- Apr-May 封锁后段方向准确率：39.1% → **69.6%**

---

### 3.5 【轻微】train_model.py 编码 Bug

**原版问题**：训练完成时打印 `✓` 符号，Windows 中文终端（GBK编码）无法处理，抛出 `UnicodeEncodeError` 导致训练日志截断。

**修复**：替换为 `OK` 字符串。

## 四、第二层模型优化

### 4.1 历史事件库扩充：15 → 17 个事件

原版15个事件偏向供应中断类（正收益），缺少贸易政策需求冲击类（负收益）的参照案例。新增：

| 新增事件 | 类型 | typical_30d | 用途 |
|---------|------|------------|------|
| 美中贸易战冲击（2018-09） | macro | -12% | 贸易政策引发需求下调的历史参照 |
| 俄乌后地缘溢价消退（2022-06） | supply_policy | -22% | 地缘高峰后价格回调的历史参照 |

**效果**：特朗普关税期间情景匹配能找到"美中贸易战"这个负收益参照，不再仅匹配到正收益的地缘政治事件。

### 4.2 霍尔木兹封锁事件参数更新

| 参数 | 原版 | 改进版 |
|------|------|--------|
| end 日期 | 2026-03-15 | 2026-05-31（封锁持续中） |
| typical_30d | 0.35 | 0.45（基于实测：2026-02-16起30日实际涨幅+52%） |
| description | 简要描述 | 更新含WTI$63→$114峰值信息 |

### 4.3 移除噪声触发条件

原版中引入 `gdelt_ci < -6.5` 单独触发条件（无需 geo_flag），测试发现该条件在验证集内额外触发136天，平均目标收益仅+0.9%，信噪比极低。改进版移除此条件，减少无效的第二层激活。

## 五、回测展示重构

### 5.1 综合输出替代原始双线展示

**原版**：回测图展示 `pred_baseline_mid`（蓝线）和 `pred_enhanced_mid`（灰虚线），两条都是 XGBoost 原始输出，封锁期不经任何处理。

**改进版**：引入 `get_integrated_output()` 计算 `integrated_df`，回测图展示 `integrated_mid`（绿线）：
- 正常期：`baseline_mid`（验证集方向准确率57%）
- 极端期：第二层情景匹配输出

同时新增：
- 绿色阴影：`integrated_low/high` 综合风险区间
- 红色背景：`extreme_active=True` 的极端期着色
- 灰色竖线：训练截止日（2026-01-01），明确标注 in-sample vs OOS 边界

### 5.2 危机窗口下拉新增真实 OOS 事件

**原版**：三个下拉选项（俄乌冲突、以哈冲突、特朗普就职）全部在训练集内，展示的是模型对已见数据的拟合，无法验证 OOS 性能。

**改进版**：新增"霍尔木兹封锁（2026-02-16）"作为第一选项，使用 `integrated_df` 展示双层架构在真实 OOS 事件上的实际表现，包含极端期红色背景。

### 5.3 绩效指标概览面板

新增顶部指标卡片（动态计算，非硬编码）：

| 指标 | 验证集OOS | 真实OOS 2026+ |
|------|---------|--------------|
| 方向准确率 | **58.4%** | 57.6% |
| MAE | 4.71% | 10.06% |
| P10-P90 覆盖率 | **93.6%** | **83.7%** |
| 信号准确率 | 55.5% | 49.4% |
| RMSE | 6.84% | 18.68% |

*注：原版报告测试集覆盖率76%（285条，2024-07~2026-01）；改进版验证集93.6%（298条，2024-10~2025-12），两者范围不同，数字不可直接对比。改进版覆盖率高的原因：极端期触发修复后第二层正确扩宽区间。*

### 5.4 移除最大回撤指标

原版无此指标。改进版曾计算"纯机械跟单策略的最大回撤"（-76%），但该指标无金融逻辑——企业银行不会100%仓位跟单，已从界面移除。

### 5.5 近60日走向图改用综合输出

**原版**：风险预测页近60日走向图使用 `pred_df`（XGBoost原始输出），封锁期显示看空（均值回归）。

**改进版**：改用 `integrated_df`，封锁期正确调用第二层，绿线方向与实际一致，区间宽度也更合理。

## 六、数据层扩展

### 6.1 回测展示延伸到今日

**原版**：`feature_matrix.csv` 最后10行因目标变量未知而截断，回测图和预测图均止于10个交易日前。

**改进方案**：新增 `recent_features.csv`（无目标变量，仅特征），`load_assets()` 启动时自动合并，回测图预测线（绿线）延伸至今日，目标线（红线）在有效数据处自然截止。

*设计原则*：训练管道不变（`feat.iloc[:-10]` 保持防泄露），展示管道与训练管道解耦。

### 6.2 历史情感数据扩展

**原版**：`daily_sentiment.csv` 仅含2026-03-24起的43天数据，训练集内2020-2025年情感特征全部为0，导致情感因子重要性实为0（与文档宣称的Top10矛盾）。

**改进过程**：
1. 使用 MediaCloud 按月粒度抓取2022-2026年石油相关新闻：3708条，独立日期587天
2. 对所有文章运行 DeepSeek 情感分析（keep_days=0，保留全量历史）
3. 生成 `daily_sentiment.csv`：647天，2022-01~2026-05

**覆盖率变化**：

| 年份 | 原版覆盖 | 改进后覆盖（交易日） |
|------|---------|------------------|
| 2022 | 0% | 33% |
| 2023 | 0% | 34% |
| 2024 | 0% | 23% |
| 2025 | 0% | 37% |

**结论**：即使覆盖率提升至30-37%，Enhanced 方向准确率（51.68%）仍低于 Baseline（57%），因为 XGBoost 无法从"67%为零"的稀疏特征中学到稳定规律。情感特征的模型重要性仍为0。正确定位：**LLM情感是实时预测增强信号，而非训练信号**。

## 七、新增功能模块

### 7.1 SHAP 因子贡献解释

**原版**：仅有 XGBoost 全局 gain 重要性柱状图（说明"整体上哪个特征最重要"）。

**改进版**：引入 `shap_explain.py`（基于 XGBoost 原生 `pred_contribs`，无需额外安装 shap 包）：
- `shap_current.csv`：当日预测的各因子贡献（正值=推高预测，负值=拉低预测）
- `shap_matrix.csv`：近90日 Top16 特征的 SHAP 时间序列热力图
- `shap_group_matrix.csv`：按因子组（价格动量/宏观金融/GDELT/航运/情感）的聚合贡献

在风险预测页特征重要性图后自动展示，CSV 文件由每日更新脚本自动生成。

**金融意义**：从"模型整体上看重什么"升级为"今天的预测由什么驱动"，直接支撑因果链解释。

### 7.2 ShipFinder/VesselAPI 双源 AIS 实时航运

**原版**：仅接入 AISStream.io 单源（需要 WebSocket 长连接，独立手动运行）。

**改进版**：`shipping_sources.py` 接入 ShipFinder + VesselAPI 双源：
- 覆盖5个咽喉点（霍尔木兹、曼德海峡、苏伊士运河、马六甲、博斯普鲁斯）
- 双源互备：ShipFinder 可用则优先，否则回退 VesselAPI
- API Key 未配置时自动跳过，不影响现有流程
- 输出格式与 `ais_snapshot.json` 完全兼容

## 八、每日更新流程改进

**原版 run_update() 调用链**：
```
油价 → 宏观 → GDELT → 新闻API → 新闻RSS → 情感分析 → PortWatch → 产油国 → 特征矩阵
```

**改进后调用链**：
```
（同原版）→ 特征矩阵 → update_shipping()  → update_shap()
                       ↑                    ↑
              ShipFinder/VesselAPI    TreeSHAP CSV 自动更新
```

两个新增步骤均有 `try/except` 保护，任一失败不影响主流程。
